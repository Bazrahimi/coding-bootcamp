CSS Reset

The purpose of CSS reset is to override or neutralize the default style of CSS which cna vary between browser. by reseting the default style, developers can apply their own styles consistently
by reseting the styles; it helps to avoid variation and inconsistencies.
Cascading Style Sheet (CSS), the term 'Cascading' refer to the process of combining and applying multiple style to an element.

the Cascading is based on three main princiles

  1. Cascade: The cascade refers to the process of how styles are applied to elements based on various factors.
    1. Style Rule and Declarations:
      • a style rule has a selector and one or more Declarations.
      • a selector target one or more elements.
      • Declarations define the specific styles to be applied to the selected elements. each Declarations consist of a CSS Property and CSS Value.
    2. Order of Style Rules:
      • In CSS file, style rule are written one after another.
      • the case check the order of style rule, to resolve conflict.
      • if multiple rule target the same element, the last rule will take precedence.
    3. Importance:
      • the '!important' Declarations can be added to style rule to give the hightest Priority.
      • when a stile rule is declared !important it will override any conflicting styles, regardless of specificity or order.
    4. Origin of Styles:
      • style can come from different source: user agent stylesheet(default stylesheet from browser), author stylesheet (the developer), user stylesheet (applied on the web browser based on user preferences and setting), and inline styles.
  2. Specificity: the browser uses to decide which property value is to applied to the element if the same element is selected multiple time by different selectors.
    The hierarchy of specificity, from highest to lowest.
    1. Inline Style: has the hightiest specificity, because they are applied directly to the element.
    2. ID Selector: it is more specific than class selector.
    3. Class Selector: class selector, attribute selector, pseudo-class has equal specificity. they target elements based on their class name.
    4. Element and Pseudo-Element Selector has lower specificity compare to class selector, it target elements by HTML element categories.
    5. Universal and Combinator Selector has the lowest specificity.
      Universal Selector target all elements by "*".
      Combinator Selector: select elements based on their relationship to other elements.
      Descendant Combinator is selected by a (space).
      Child Combinator is selected by ">".
      Adjacent Sibling Combinator is selected by "+". these selector has the lowest specificity.

  3. Inheritance: allow certain styles to be inherited by child elements from their parent elements. when a style element is applied to the parent, it will automatically applied to child or nested element too. this help reducing the amount CSS code required.
    Some commonly elements which are used to have child or nested elements in them.


    Empty Elements or Void Elements are not able to have child element nested in them.
