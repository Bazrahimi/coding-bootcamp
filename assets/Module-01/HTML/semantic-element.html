Semantic HTML Elements

"Semantic" comes from the Greek word "Semantikos" and translates to "significant" or "meaningful".
The advantages of semantic elements are that they carry meaning, convey the purpose and structure of HTML elements, which improves the SEO, accessibility, and maintainability of the webpage.

  • header: Represents the introduction and navigation of the webpage.

  • nav: Defines a section of navigation links. For example, on the above Student Bio webpage, where students are required to insert links for emails, GitHub, and portfolio, it is best practice to put the multiple links content in "nav" elements.

  • main: Indicates the main content of the webpage. It improves the accessibility of the webpage as screen readers are able to use the main element to identify and navigate to the main content of the page easily.
    SEO prioritizes the main content when indexing or ranking the webpage.
    Styling and Layout: You can specify a specific style and layout for the main element.

  • article: Improves accessibility and SEO by considering it as a source of information and potentially improving its visibility and ranking in search engines.

  • section: Defines a section of the webpage.
    Document Structure: Organizes and structures the content of the HTML.
    Accessibility.
    Styling and Layout.

  • aside: Represents content related to the main content. It can be sidebars or pull quotes.

  • footer: Represents the footer of the webpage. It contains metadata or contact details.

  • figure: Represents self-contained content (standalone), such as an image or illustration.
